
Purpose
This test case verifies the reintroduction of the Alt-Enter keyboard shortcut to access the properties of a specific bookmark.
Initial Conditions
A set of bookmarks must already exist in the Bookmarks Manager.
The browser must be the active window.
Steps/Description
Open the Bookmarks Manager (Library).
Navigate to a specific bookmark entry.
Highlight the bookmark by clicking on it once.
Press the Alt + Enter keys simultaneously.
Expected Results
The "Properties" dialog box for the selected bookmark must open immediately.
The dialog should display fields for Name, Location, Keyword, and Description.
Purpose
This test case verifies the "Bookmark This Group of Tabs" feature, ensuring it functions as a distinct entity from standard folders as per the Mozilla 1.7.12 behavior.
Initial Conditions
Multiple tabs (at least three) must be open with different URLs.
The user must be on a version of the browser where this feature is implemented.
Steps/Description
Right-click on any open tab or select the "Bookmarks" menu.
Select the option "Bookmark This Group of Tabs".
In the resulting dialog, observe the default view (file tree) and enter a name for the group.
Save the group to the Bookmarks Toolbar.
Perform a single click on the newly created bookmark group.
Expected Results
The "Bookmark This Group of Tabs" dialog should default to a file tree view for quick saving.
The resulting bookmark item should have a distinct visual icon/appearance compared to standard folders.
A single click on the group must trigger all contained URLs to open in tabs simultaneously.
Purpose
This test case verifies the creation and dynamic population of "Smart Folders" based on user-defined tags and criteria.
Initial Conditions
A collection of bookmarks exists.
Bookmarks are tagged with various labels (e.g., "News", "Games", "Ottawa").
Some bookmarks are "Live Bookmarks" (RSS feeds).
Steps/Description
Create a new "Smart Folder" via the Bookmark Manager.
Define the criteria for the folder: Tag = "News" AND Tag = "Ottawa".
Save the folder and observe the contents.
Create a second Smart Folder with the criteria: Type = "Live Bookmark".
Add a new bookmark with the tags "News" and "Ottawa" while the Smart Folder is active.
Expected Results
The first Smart Folder must automatically display only bookmarks that contain both the "News" and "Ottawa" tags.
The second Smart Folder must display all Live Bookmarks regardless of their tagging.
The Smart Folder must update dynamically and display the newly added bookmark without requiring a manual refresh.
Purpose
This test case verifies that the "Add Bookmark" dialog includes fields for tags and comments and that this data is searchable.
Initial Conditions
The browser is open to a live website.
The Bookmarks Sidebar is open.
Steps/Description
Press Ctrl+D (or the "Add Bookmark" command).
In the "Add Bookmark" dialog, locate the "Tags" and "Description/Comments" fields.
Enter a unique tag (e.g., "AlphaTest") and a unique comment (e.g., "Testing metadata search").
Save the bookmark.
In the Sidebar search box, type "AlphaTest".
Clear the search and type "Testing metadata".
Expected Results
The "Add Bookmark" dialog must contain explicit input fields for both Tags and Description.
Searching for the tag in the sidebar must return the specific bookmark.
Searching for words within the comment/description field must return the specific bookmark.
Purpose
This test case verifies the ability to assign a keyword to a folder and trigger the "Open in Tabs" command via the address bar.
Initial Conditions
A bookmark folder exists containing at least five bookmarks.
The folder is located within the Bookmarks Menu or Toolbar.
Steps/Description
Right-click the folder and select "Properties".
In the "Keyword" field, enter a unique shortcut (e.g., "dailyread").
Save the properties.
Navigate to the browser address bar.
Type "dailyread" and press Enter.
Expected Results
The browser must recognize the keyword as a folder command.
All bookmarks contained within that folder must open simultaneously in new tabs.
Purpose
This test case verifies that Live Bookmarks visually differentiate between "read" and "unread" items and support a "Next Unread" function.
Initial Conditions
A Live Bookmark (RSS feed) is added to the toolbar.
The feed contains several items, some of which have been clicked (visited) and some which have not.
Steps/Description
Open the Live Bookmark folder.
Observe the visual style of the items.
Click on one "unread" item to open it.
Re-open the Live Bookmark folder and observe the item again.
Click the "Next Unread" button on the toolbar (if implemented).
Expected Results
Unread items must be visually distinct (e.g., bold text or a specific icon).
Once an item is visited, its CSS state must change to the :visited pseudo-class (e.g., normal weight or grayed out).
The "Next Unread" button must automatically navigate the browser to the next item in the feed that has not been visited.
Purpose
This test case verifies the "Duplicate Bookmark Warning" functionality to prevent redundant entries.
Initial Conditions
A bookmark for http://www.example.com already exists in the "Work" folder.
Steps/Description

Click the "Add Bookmark" star or menu item.

Purpose
This test case verifies the "Temporary/Time Based Bookmarks" functionality where bookmarks expire after a set duration.
Initial Conditions
A folder named "Temp" is created.
The folder properties are set to "Expire items after 24 hours".
Steps/Description
Add a bookmark to the "Temp" folder at 12:00 PM on Day 1.
Verify the bookmark is visible and accessible.
Advance the system clock to 1:00 PM on Day 2 (or wait 25 hours).
Open the "Temp" folder and check for the bookmark.
Expected Results
The bookmark must be automatically deleted or hidden once the 24-hour threshold is crossed.
The folder should reflect that it no longer contains the expired item.
Purpose
This test case verifies the "Relative Bookmarks" functionality for portability using the profile:// protocol.
Initial Conditions
The browser is running a specific user profile.
A file named notes.txt exists inside the profile folder.
Steps/Description
Create a new bookmark.
In the "Location" field, enter profile://notes.txt.
Save the bookmark.
Click the bookmark to open it.
Expected Results
The browser must resolve profile:// to the local path of the current active profile.
The notes.txt file must open successfully in the browser or associated application.
