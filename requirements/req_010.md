# req_010 — Média dos elementos acima de um limiar

## Descrição

O sistema deve calcular a **média aritmética** considerando **apenas** os
elementos de `values` que são **estritamente maiores** que `threshold`. Elementos
iguais ao limiar **não** entram no cálculo. Quando nenhum elemento qualifica (ou
a lista está vazia), o resultado é `0.0`.

## Interface

- **Módulo a importar:** `solution`
- **Assinatura:** `average_above(values: list, threshold: float) -> float`

```python
from solution import average_above
```

## Domínio de entrada

- `values`: lista de números (`int` ou `float`). Pode estar vazia.
- `threshold`: número (`int` ou `float`). Pode ser negativo.

## Regras

1. Considere apenas os elementos `x` de `values` tais que `x > threshold`
   (**estritamente** maior; `x == threshold` é descartado).
2. O resultado é a média aritmética dos elementos qualificados:
   `soma(qualificados) / quantidade(qualificados)`.
3. Se **nenhum** elemento for `> threshold` (incluindo `values` vazia), o
   resultado é `0.0`.
4. O resultado é **arredondado para 2 casas decimais** usando o `round()` nativo
   do Python (*round half to even*, banker's rounding). Ex.: `round(2.675, 2)`
   segue a semântica do float nativo.

## Classes de equivalência

- `values` vazia → `0.0`;
- nenhum elemento `> threshold` → `0.0`;
- alguns elementos `> threshold` → média desses elementos;
- todos os elementos `> threshold` → média de toda a lista;
- elemento `== threshold` presente → ele é ignorado.

## Valores-limite

- `average_above([10], 10)` → `0.0` (igual ao limiar não conta);
- `average_above([10.0001], 10)` → `10.0` (estritamente maior, único qualificado);
- `average_above([], 5)` → `0.0` (lista vazia);
- `average_above([5, 5, 5], 5)` → `0.0` (nenhum estritamente maior).

## Entradas inválidas

- `values` vazia é tratada como caso definido (retorna `0.0`), não erro.
- Não há tratamento especial para tipos não numéricos: assume-se que `values`
  contém apenas números e `threshold` é numérico (fora de escopo validar tipos).

## Invariantes

- O resultado é a média de um subconjunto de `values`; portanto, quando há
  qualificados, ele fica entre o menor e o maior dos qualificados.
- O resultado nunca depende de elementos `<= threshold`.

## Critérios de aceitação

- AC1. `average_above([1, 2, 3, 4], 2) == 3.5` (média de 3 e 4).
- AC2. `average_above([10], 10) == 0.0` (igual ao limiar é descartado).
- AC3. `average_above([], 5) == 0.0` (lista vazia).
- AC4. `average_above([5, 5, 5], 5) == 0.0` (nenhum estritamente maior).
- AC5. `average_above([1.111, 2.222, 3.333], 0) == 2.22` (arredondamento para 2 casas).
