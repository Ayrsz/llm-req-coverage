# req_014 — Saldo final após sequência de operações

## Descrição

O sistema deve calcular o saldo final de uma conta a partir de um saldo inicial
e de uma sequência ordenada de operações. Cada operação é um depósito (soma) ou
um saque (subtrai), aplicada na ordem em que aparece. Um saque só é efetivado se
houver **saldo suficiente**; caso contrário a operação é **ignorada** (o saldo
nunca fica negativo).

## Interface

- **Módulo a importar:** `solution`
- **Assinatura:** `final_balance(initial: float, operations: list) -> float`

```python
from solution import final_balance
```

## Domínio de entrada

- `initial`: número (`int` ou `float`) em reais. Representa o saldo inicial
  (assume-se `>= 0`).
- `operations`: lista de tuplas `(tipo, valor)`, aplicadas em ordem.
  - `tipo`: string. Valores reconhecidos: `"deposit"` e `"withdraw"`.
    Qualquer outro valor é um tipo desconhecido.
  - `valor`: número (`int` ou `float`) `>= 0` movimentado pela operação.
- Lista vazia: nenhuma operação aplicada, retorna o próprio `initial`.

## Regras

1. O saldo começa em `initial`.
2. Para cada operação `(tipo, valor)`, na ordem da lista:
   - `"deposit"`: soma `valor` ao saldo;
   - `"withdraw"`: subtrai `valor` do saldo **somente se** `valor <= saldo`
     (saldo suficiente); caso contrário a operação é **ignorada** (saldo
     inalterado);
   - tipo **desconhecido**: a operação é **ignorada** (saldo inalterado).
3. Invariante: o saldo nunca fica negativo.
4. O resultado é **arredondado para 2 casas decimais** ao final.

## Classes de equivalência

- lista vazia → retorna `initial`;
- só depósitos → soma de todos os valores ao inicial;
- saque com saldo suficiente → efetivado (subtrai);
- saque com saldo insuficiente → ignorado (saldo inalterado);
- operação de tipo desconhecido → ignorada;
- mistura de operações → aplicadas em ordem.

## Valores-limite

- `withdraw` de `valor == saldo` → permitido, saldo vai a `0.0` (fronteira **fechada**);
- `withdraw` de `valor == saldo + 0.01` → ignorado (insuficiente);
- `withdraw` de `valor == 0` → permitido, saldo inalterado;
- lista vazia → `initial`.

## Entradas inválidas

- Tipo de operação fora de `{"deposit", "withdraw"}`: ignorado (não levanta erro).
- Saque que tornaria o saldo negativo: ignorado (não levanta erro).

## Invariantes

- O saldo final nunca é negativo.
- Operações ignoradas não alteram o saldo.
- A ordem das operações é respeitada.

## Critérios de aceitação

- AC1. `final_balance(100.0, []) == 100.0` (lista vazia retorna inicial).
- AC2. `final_balance(100.0, [("deposit", 50.0)]) == 150.0` (depósito soma).
- AC3. `final_balance(100.0, [("withdraw", 30.0)]) == 70.0` (saque com saldo suficiente).
- AC4. `final_balance(100.0, [("withdraw", 150.0)]) == 100.0` (saque insuficiente ignorado).
- AC5. `final_balance(100.0, [("withdraw", 100.0)]) == 0.0` (fronteira: saque igual ao saldo, vai a 0).
- AC6. `final_balance(100.0, [("bonus", 50.0)]) == 100.0` (tipo desconhecido ignorado).
- AC7. `final_balance(100.0, [("deposit", 50.0), ("withdraw", 200.0), ("withdraw", 100.0)]) == 50.0` (ordem; 1º saque ignorado, 2º efetivado).
