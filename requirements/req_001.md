# req_001 — Cálculo de preço final com desconto

## Descrição

O sistema deve calcular o valor final de uma compra aplicando desconto conforme
o tipo de cliente. O desconto só se aplica a compras **acima** de R$ 100,00.

## Interface

- **Módulo a importar:** `solution`
- **Assinatura:** `calculate_final_price(value: float, customer_type: str) -> float`

```python
from solution import calculate_final_price
```

## Domínio de entrada

- `value`: número (`int` ou `float`) em reais. Pode ser negativo (entrada inválida).
- `customer_type`: string. Valores reconhecidos: `"common"` e `"premium"`.
  Qualquer outro valor é tratado como `"common"` (cliente sem benefício especial).

## Regras

1. Se `value` for negativo, o preço final é `0.0` (invariante: nunca negativo).
2. Se `value <= 100`, não há desconto: o preço final é o próprio `value`.
3. Se `value > 100`:
   - cliente `"premium"`: 10% de desconto (`value * 0.90`);
   - demais clientes: 5% de desconto (`value * 0.95`).
4. O resultado é **arredondado para 2 casas decimais usando arredondamento
   meio-para-cima** (*round half up*): no empate de meio-centavo, arredonda para
   cima. Ex.: `142.975 → 142.98`, `95.015 → 95.02`.

## Classes de equivalência

- valor negativo → `0.0`;
- `0 <= value <= 100` → sem desconto (qualquer tipo de cliente);
- `value > 100` + `common` → 5%;
- `value > 100` + `premium` → 10%;
- `customer_type` desconhecido → tratado como `common`.

## Valores-limite

- `value = 100` → sem desconto (limite **fechado**, não recebe desconto);
- `value = 100.01` → recebe desconto;
- `value = 0` → `0.0`;
- `value = -0.01` → `0.0`.

## Invariantes

- O preço final nunca é negativo.
- O preço final nunca é maior que `value` (para `value >= 0`).

## Precisão

O arredondamento é **meio-para-cima** (*round half up*), determinístico — não
use o `round()` nativo do Python (que arredonda meio-para-par sobre o float). A
implementação usa `Decimal.quantize(..., ROUND_HALF_UP)`. Ao comparar floats nos
testes, use `pytest.approx`.

## Critérios de aceitação

- AC1. `calculate_final_price(100, "common") == 100.0` (limite, sem desconto).
- AC2. `calculate_final_price(200, "common")` ≈ `190.0` (5%).
- AC3. `calculate_final_price(200, "premium")` ≈ `180.0` (10%).
- AC4. `calculate_final_price(-50, "common") == 0.0` (invariante).
- AC5. `calculate_final_price(200, "desconhecido")` ≈ `190.0` (tratado como common).
