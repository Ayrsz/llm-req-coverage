# req_012 — Imposto de renda progressivo por faixas

## Descrição

O sistema deve calcular o imposto de renda devido sobre um rendimento mensal,
aplicando **alíquotas progressivas (marginais) por faixa**. Cada faixa tributa
**apenas a parcela do rendimento que cai dentro dela**, não o rendimento total a
uma alíquota única.

## Interface

- **Módulo a importar:** `solution`
- **Assinatura:** `income_tax(income: float) -> float`

```python
from solution import income_tax
```

## Domínio de entrada

- `income`: número (`int` ou `float`) em reais, representando o rendimento
  mensal. Pode ser negativo (entrada inválida).

## Faixas e alíquotas

São quatro faixas progressivas. Cada faixa incide **somente sobre a parte do
rendimento dentro do intervalo**:

| Faixa | Intervalo (R$)        | Alíquota marginal |
| ----- | --------------------- | ----------------- |
| 1     | `0` até `2000`        | 0% (isento)       |
| 2     | acima de `2000` até `3000` | 7.5%         |
| 3     | acima de `3000` até `4500` | 15%          |
| 4     | acima de `4500`       | 22.5%             |

**Fronteiras (qual lado é incluso).** O limite superior de cada faixa é
**fechado** (incluso na faixa de baixo): um rendimento de exatamente `2000`
permanece isento; exatamente `3000` é tributado só na faixa 2; exatamente `4500`
é tributado nas faixas 2 e 3. Apenas a parcela **estritamente acima** do limite
sobe para a próxima faixa.

## Regras

1. Se `income` for negativo, o imposto é `0.0` (invariante: nunca negativo).
2. A parcela de `0` a `2000` é isenta (contribui `0.0` de imposto).
3. A parcela **acima de** `2000` e **até** `3000` é tributada a 7.5%
   (no máximo `1000 * 0.075 = 75.0`).
4. A parcela **acima de** `3000` e **até** `4500` é tributada a 15%
   (no máximo `1500 * 0.15 = 225.0`).
5. A parcela **acima de** `4500` é tributada a 22.5%.
6. O imposto total é a **soma** das parcelas de cada faixa.
7. O resultado é **arredondado para 2 casas decimais usando arredondamento
   meio-para-cima** (*round half up*): no empate de meio-centavo, arredonda para
   cima.

## Classes de equivalência

- rendimento negativo → `0.0`;
- `0 <= income <= 2000` → imposto `0.0` (isento);
- `2000 < income <= 3000` → só faixa 2 (7.5% da parte acima de 2000);
- `3000 < income <= 4500` → faixas 2 + 3;
- `income > 4500` → faixas 2 + 3 + 4.

## Valores-limite

- `income = 2000` → `0.0` (limite **fechado** na faixa isenta);
- `income = 2000.01` → tributado (faixa 2 começa acima de 2000);
- `income = 3000` → `75.0` (apenas faixa 2 cheia);
- `income = 4500` → `75.0 + 225.0 = 300.0` (faixas 2 e 3 cheias);
- `income = 0` → `0.0`;
- `income = -0.01` → `0.0`.

## Entradas inválidas

- `income < 0` → resultado `0.0` (não lança exceção).

## Invariantes

- O imposto nunca é negativo.
- O imposto nunca é maior que `income` (para `income >= 0`).
- O imposto é **monotônico não-decrescente** em `income` (mais renda nunca
  reduz o imposto).
- A alíquota efetiva nunca atinge 22.5% (sempre há faixas isenta/baixas abaixo).

## Precisão

O arredondamento é **meio-para-cima** (*round half up*), determinístico — não
use o `round()` nativo do Python (que arredonda meio-para-par sobre o float). A
implementação usa `Decimal.quantize(..., ROUND_HALF_UP)`. Ao comparar floats nos
testes, use `pytest.approx`.

## Critérios de aceitação

- AC1. `income_tax(2000) == 0.0` (limite fechado, isento).
- AC2. `income_tax(3000)` ≈ `75.0` (faixa 2 cheia: `1000 * 0.075`).
- AC3. `income_tax(4500)` ≈ `300.0` (faixas 2 + 3: `75.0 + 225.0`).
- AC4. `income_tax(5000)` ≈ `412.5` (`75.0 + 225.0 + 500 * 0.225`).
- AC5. `income_tax(-100) == 0.0` (invariante, entrada inválida).
- AC6. `income_tax(2500)` ≈ `37.5` (só faixa 2: `500 * 0.075`).
