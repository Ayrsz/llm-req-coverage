# req_011 — Item mais frequente com desempate por primeira ocorrência

## Descrição

O sistema deve retornar a **string que mais aparece** em `items`. Em caso de
empate de frequência, vence a string que **aparece primeiro** em `items` (menor
índice de primeira ocorrência). Lista vazia retorna `""`.

## Interface

- **Módulo a importar:** `solution`
- **Assinatura:** `most_frequent(items: list) -> str`

```python
from solution import most_frequent
```

## Domínio de entrada

- `items`: lista de strings. Pode estar vazia. Strings podem se repetir.

## Regras

1. Conte quantas vezes cada string distinta aparece em `items`.
2. O resultado é a string com a **maior contagem**.
3. **Desempate:** se duas ou mais strings têm a maior contagem, retorna aquela
   cuja **primeira ocorrência** acontece no menor índice de `items` (a que
   "aparece primeiro" ao percorrer a lista da esquerda para a direita). A ordem
   de inserção das chaves é a ordem de primeira aparição.
4. Se `items` for vazia, retorna `""`.

## Classes de equivalência

- `items` vazia → `""`;
- um vencedor isolado (frequência estritamente maior) → essa string;
- empate de frequência entre duas ou mais → a de primeira ocorrência mais cedo;
- todos os elementos distintos (todos com frequência 1) → o primeiro elemento.

## Valores-limite

- `most_frequent([])` → `""` (lista vazia);
- `most_frequent(["a"])` → `"a"` (único elemento);
- `most_frequent(["a", "b"])` → `"a"` (empate 1×1; "a" aparece primeiro);
- `most_frequent(["b", "a", "a", "b"])` → `"b"` (empate 2×2; "b" aparece primeiro).

## Entradas inválidas

- `items` vazia é caso definido (retorna `""`), não erro.
- Assume-se que os elementos são strings (fora de escopo validar tipos).

## Invariantes

- O resultado é sempre um elemento de `items` (ou `""` se vazia).
- A contagem do resultado é `>=` à contagem de qualquer outro elemento.

## Exemplos de desempate

- `["x", "y", "x", "y"]` → `"x"` (ambos com 2; "x" aparece no índice 0).
- `["y", "x", "y", "x"]` → `"y"` (ambos com 2; "y" aparece no índice 0).
- `["c", "b", "a"]` → `"c"` (todos com 1; "c" é o primeiro).

## Critérios de aceitação

- AC1. `most_frequent(["a", "b", "a"]) == "a"` (vencedor isolado).
- AC2. `most_frequent([]) == ""` (lista vazia).
- AC3. `most_frequent(["a", "b"]) == "a"` (empate; primeira ocorrência).
- AC4. `most_frequent(["b", "a", "a", "b"]) == "b"` (empate; "b" aparece primeiro).
- AC5. `most_frequent(["x"]) == "x"` (único elemento).
