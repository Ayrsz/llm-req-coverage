# req_008 — Ano bissexto (calendário gregoriano)

## Descrição

O sistema deve determinar se um ano é **bissexto** segundo a regra do calendário
gregoriano. Um ano é bissexto se for divisível por 4 **e** (não divisível por 100
**ou** divisível por 400).

## Interface

- **Módulo a importar:** `solution`
- **Assinatura:** `is_leap_year(year: int) -> bool`

```python
from solution import is_leap_year
```

## Domínio de entrada

- `year`: número inteiro. O domínio válido são anos positivos (`year >= 1`).
  `year <= 0` é tratado como **entrada inválida**.

## Regras

1. Se `year <= 0`, a entrada é inválida e o resultado é `False`
   (decisão documentada: ano não positivo não é bissexto).
2. Se `year` é divisível por 400 → bissexto (`True`).
3. Senão, se `year` é divisível por 100 → **não** bissexto (`False`).
4. Senão, se `year` é divisível por 4 → bissexto (`True`).
5. Caso contrário → **não** bissexto (`False`).

Equivalente lógico das regras 2–5: bissexto sse
`year % 4 == 0 and (year % 100 != 0 or year % 400 == 0)`.

## Classes de equivalência

- `year <= 0` → `False` (inválido);
- divisível por 400 → `True` (ex.: 2000);
- divisível por 100 mas não por 400 → `False` (ex.: 1900, 2100);
- divisível por 4 mas não por 100 → `True` (ex.: 2004);
- não divisível por 4 → `False` (ex.: 2001).

## Valores-limite

- `year = 0` → `False` (limite inválido);
- `year = 1` → `False` (não divisível por 4);
- `year = 4` → `True` (menor positivo divisível por 4);
- `year = 1900` → `False` (divisível por 100, não por 400);
- `year = 2000` → `True` (divisível por 400);
- `year = 2004` → `True` (divisível por 4, não por 100);
- `year = 2100` → `False` (divisível por 100, não por 400).

## Entradas inválidas

- Qualquer `year <= 0` (zero ou negativo) retorna `False`. A função nunca
  levanta exceção para inteiros: o tipo de retorno é sempre `bool`.

## Invariantes

- A saída é sempre um `bool` (`True` ou `False`), nunca `None` nem exceção.
- Para todo `year` divisível por 400 e positivo, a saída é `True`.
- Para todo `year` positivo divisível por 100 mas não por 400, a saída é `False`.

## Critérios de aceitação

- AC1. `is_leap_year(2000) is True` (divisível por 400).
- AC2. `is_leap_year(1900) is False` (divisível por 100, não por 400).
- AC3. `is_leap_year(2004) is True` (divisível por 4, não por 100).
- AC4. `is_leap_year(2100) is False` (divisível por 100, não por 400).
- AC5. `is_leap_year(2001) is False` (não divisível por 4).
- AC6. `is_leap_year(0) is False` e `is_leap_year(-4) is False` (inválidos).
