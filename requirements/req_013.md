# req_013 — Custo de frete por região, peso e modalidade

## Descrição

O sistema deve calcular o custo de frete de uma encomenda a partir de uma
**tarifa base por região**, um **adicional por peso excedente** e um possível
**acréscimo de modalidade expressa**.

## Interface

- **Módulo a importar:** `solution`
- **Assinatura:** `shipping_cost(weight: float, region: str, express: bool) -> float`

```python
from solution import shipping_cost
```

## Domínio de entrada

- `weight`: número (`int` ou `float`) em quilogramas. Pode ser `<= 0` (inválido).
- `region`: string com o nome da região. Valores reconhecidos:
  `"sul"`, `"sudeste"`, `"norte"`, `"nordeste"`, `"centro-oeste"`.
  Qualquer outro valor é tratado como **região desconhecida** (base padrão).
- `express`: booleano. `True` aplica o acréscimo de modalidade expressa.

## Tarifas base por região

| Região                                  | Base (R$) |
| --------------------------------------- | --------- |
| `"sul"`, `"sudeste"`                    | `10.0`    |
| `"norte"`, `"nordeste"`, `"centro-oeste"` | `20.0`  |
| região desconhecida (qualquer outra)    | `25.0`    |

A comparação de `region` é **literal** (case-sensitive); por exemplo `"Sul"`
não é reconhecida e cai na base padrão `25.0`.

## Regras

1. Se `weight <= 0`, o custo é `0.0` (entrada inválida; invariante: nunca negativo).
2. O custo parte da **tarifa base** da região (ver tabela; desconhecida → `25.0`).
3. **Adicional por peso:** para cada quilograma **acima de `5` kg** soma-se
   `2.0` por kg. A fronteira é **fechada em 5**: peso de exatamente `5` kg **não**
   gera adicional. O excedente é `max(0, weight - 5)` e o adicional é
   `excedente * 2.0` (proporcional, não por kg inteiro).
4. **Modalidade expressa:** se `express` for `True`, o subtotal (base +
   adicional de peso) recebe **50% de acréscimo** (multiplica por `1.5`).
5. O resultado é **arredondado para 2 casas decimais usando arredondamento
   meio-para-cima** (*round half up*).

## Classes de equivalência

- `weight <= 0` → `0.0` (qualquer região/modalidade);
- região conhecida barata (`sul`/`sudeste`) → base `10.0`;
- região conhecida cara (`norte`/`nordeste`/`centro-oeste`) → base `20.0`;
- região desconhecida → base `25.0`;
- `weight <= 5` → sem adicional de peso;
- `weight > 5` → adicional proporcional;
- `express=False` → sem acréscimo; `express=True` → ×1.5.

## Valores-limite

- `weight = 5`, `sul`, `express=False` → `10.0` (limite **fechado**, sem adicional);
- `weight = 5.01`, `sul`, `express=False` → `10.02` (`10 + 0.01*2`);
- `weight = 0` → `0.0`;
- `weight = -1` → `0.0`;
- `weight = 6`, `sudeste`, `express=False` → `12.0` (`10 + 1*2`).

## Entradas inválidas

- `weight <= 0` → resultado `0.0` (não lança exceção).
- `region` não reconhecida → base padrão `25.0` (não lança exceção).

## Invariantes

- O custo nunca é negativo.
- Para a mesma região e peso válido, `express=True` nunca é mais barato que
  `express=False`.
- O custo é não-decrescente em `weight` para `weight > 0`.

## Precisão

O arredondamento é **meio-para-cima** (*round half up*), determinístico — não
use o `round()` nativo do Python. A implementação usa
`Decimal.quantize(..., ROUND_HALF_UP)`. Ao comparar floats nos testes, use
`pytest.approx`.

## Critérios de aceitação

- AC1. `shipping_cost(5, "sul", False) == 10.0` (base barata, limite de peso fechado).
- AC2. `shipping_cost(5, "norte", False) == 20.0` (base cara, sem adicional).
- AC3. `shipping_cost(6, "sudeste", False)` ≈ `12.0` (`10 + 1*2`).
- AC4. `shipping_cost(6, "sul", True)` ≈ `18.0` (`(10 + 2) * 1.5`).
- AC5. `shipping_cost(3, "marte", False) == 25.0` (região desconhecida, base padrão).
- AC6. `shipping_cost(-1, "sul", True) == 0.0` (invariante, entrada inválida).
- AC7. `shipping_cost(10, "nordeste", True)` ≈ `45.0` (`(20 + 5*2) * 1.5`).
