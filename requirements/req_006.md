# req_006 — Validação de endereço IPv4

## Descrição

O sistema deve validar se uma string representa um endereço IPv4 sintaticamente
válido na sua forma canônica (*dotted-decimal*). A função retorna `True` apenas
quando **todas** as regras são satisfeitas, e `False` caso contrário.

## Interface

- **Módulo a importar:** `solution`
- **Assinatura:** `is_valid_ipv4(s: str) -> bool`

```python
from solution import is_valid_ipv4
```

## Domínio de entrada

- `s`: string. Pode ser vazia. Pode conter qualquer caractere (dígitos, pontos,
  espaços, letras, sinais). Apenas a forma canônica IPv4 é aceita.

## Regras

A string é um IPv4 **válido** (`True`) somente se satisfizer **todas** as condições:

1. há exatamente **4 octetos** separados por `"."` (exatamente 3 pontos; nenhum
   octeto vazio, ou seja, sem pontos consecutivos e sem ponto nas pontas);
2. cada octeto é composto **apenas por dígitos decimais** `0`–`9` (sem sinal, sem
   espaços, sem caracteres não numéricos);
3. cada octeto tem o valor inteiro no intervalo **`[0, 255]`** (255 válido, 256
   inválido);
4. cada octeto **não tem zeros à esquerda**: `"0"` é válido, mas `"00"`, `"01"`
   e `"007"` são inválidos (um octeto com mais de um caractere não pode começar
   por `0`).

Se qualquer regra falhar, o retorno é `False`.

## Classes de equivalência

- IPv4 canônico bem-formado (4 octetos, cada um em `[0,255]`, sem zero à esquerda) → `True`;
- número de octetos diferente de 4 (menos ou mais) → `False`;
- octeto vazio (pontos consecutivos / nas pontas) → `False`;
- octeto com caractere não-dígito (letra, espaço, sinal) → `False`;
- octeto fora de `[0,255]` → `False`;
- octeto com zero à esquerda → `False`;
- string vazia → `False`.

## Valores-limite

- `"0.0.0.0"` → `True` (mínimo);
- `"255.255.255.255"` → `True` (máximo, 255 válido);
- `"256.0.0.0"` → `False` (256 inválido);
- `"0.0.0.255"` → `True`;
- octeto `"0"` → válido; octeto `"00"` → inválido;
- `""` (vazia) → `False`.

## Entradas inválidas

- `""` → `False` (vazia);
- `"1.2.3"` → `False` (apenas 3 octetos);
- `"1.2.3.4.5"` → `False` (5 octetos);
- `"1.2.3."` → `False` (octeto vazio à direita);
- `".1.2.3"` → `False` (octeto vazio à esquerda);
- `"1..2.3"` → `False` (octeto vazio no meio);
- `"1.2.3.256"` → `False` (fora de faixa);
- `"01.2.3.4"` → `False` (zero à esquerda);
- `"1.2.3.a"` → `False` (não-dígito);
- `"1.2.3. 4"` ou `" 1.2.3.4"` → `False` (espaço);
- `"+1.2.3.4"` / `"-1.2.3.4"` → `False` (sinal não é dígito).

## Invariantes

- O retorno é sempre `bool` (`True` ou `False`), nunca lança exceção para
  nenhuma string de entrada.
- A validação é puramente sintática e determinística (não depende de DNS,
  estado externo ou tempo).

## Critérios de aceitação

- AC1. `is_valid_ipv4("0.0.0.0") is True` (mínimo).
- AC2. `is_valid_ipv4("255.255.255.255") is True` (máximo, 255 válido).
- AC3. `is_valid_ipv4("256.0.0.0") is False` (256 fora de faixa).
- AC4. `is_valid_ipv4("01.2.3.4") is False` (zero à esquerda).
- AC5. `is_valid_ipv4("00.0.0.0") is False` ("00" tem zero à esquerda).
- AC6. `is_valid_ipv4("1.2.3") is False` (apenas 3 octetos).
- AC7. `is_valid_ipv4("1.2.3.4.5") is False` (5 octetos).
- AC8. `is_valid_ipv4("1.2.3.") is False` (octeto vazio).
- AC9. `is_valid_ipv4("1.2.3.a") is False` (caractere não numérico).
- AC10. `is_valid_ipv4("") is False` (vazia).
- AC11. `is_valid_ipv4("192.168.0.1") is True` (caso típico).
