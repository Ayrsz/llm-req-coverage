# req_004 — Classificação de IMC (faixas da OMS)

## Descrição

O sistema deve calcular o Índice de Massa Corporal (IMC = `weight / height²`,
peso em kg, altura em metros) e classificá-lo nas faixas da OMS. Entradas que
não permitem um IMC bem definido (altura não positiva ou peso negativo) são
tratadas como inválidas.

## Interface

- **Módulo a importar:** `solution`
- **Assinatura:** `bmi_category(weight: float, height: float) -> str`

```python
from solution import bmi_category
```

## Domínio de entrada

- `weight`: número (`int` ou `float`) em quilogramas. Deve ser `>= 0`; valor
  negativo é entrada inválida.
- `height`: número (`int` ou `float`) em metros. Deve ser `> 0`; valor `<= 0`
  é entrada inválida (divisão por zero / sem significado físico).

## Regras

1. Se `height <= 0` **ou** `weight < 0`, retorna `"invalido"` (guarda de
   entrada, avaliada antes de qualquer cálculo).
2. Caso contrário, calcula `bmi = weight / (height * height)`.
3. Classifica `bmi`:
   - `bmi < 18.5` → `"abaixo"`;
   - `18.5 <= bmi < 25` → `"normal"`;
   - `25 <= bmi < 30` → `"sobrepeso"`;
   - `bmi >= 30` → `"obesidade"`.

## Classes de equivalência

- entrada inválida (`height <= 0` ou `weight < 0`) → `"invalido"`;
- `bmi` baixo → `"abaixo"`;
- `bmi` saudável → `"normal"`;
- `bmi` elevado → `"sobrepeso"`;
- `bmi` muito elevado → `"obesidade"`.

## Valores-limite

Cada fronteira é **fechada à esquerda** (o valor exato pertence à faixa
superior):

- `bmi = 18.5` → `"normal"` (limite inferior **incluso** em normal);
- `bmi = 24.99...` → `"normal"`; `bmi = 25` → `"sobrepeso"` (25 **incluso** em sobrepeso);
- `bmi = 29.99...` → `"sobrepeso"`; `bmi = 30` → `"obesidade"` (30 **incluso** em obesidade);
- `height = 0` → `"invalido"`;
- `weight = 0` → `bmi = 0` → `"abaixo"` (peso zero é válido, não inválido).

## Entradas inválidas

- `height <= 0` (inclui `0` e negativos) → `"invalido"`.
- `weight < 0` → `"invalido"`.
- `weight = 0` com `height > 0` é **válido** (resulta em `"abaixo"`).

## Invariantes

- O retorno é sempre uma das cinco strings:
  `{"invalido", "abaixo", "normal", "sobrepeso", "obesidade"}`.
- IMC é monotônico em `weight` (para `height` fixo): aumentar o peso nunca
  desloca a classificação para uma faixa inferior.

## Critérios de aceitação

- AC1. `bmi_category(50, 1.75) == "abaixo"` (bmi ≈ 16.33).
- AC2. `bmi_category(56.66, 1.75) == "normal"` (bmi ≈ 18.5, limite inferior).
- AC3. `bmi_category(70, 1.75) == "normal"` (bmi ≈ 22.86).
- AC4. `bmi_category(76.5625, 1.75) == "sobrepeso"` (bmi = 25.0, limite incluso).
- AC5. `bmi_category(91.875, 1.75) == "obesidade"` (bmi = 30.0, limite incluso).
- AC6. `bmi_category(70, 0) == "invalido"` (altura não positiva).
- AC7. `bmi_category(-1, 1.75) == "invalido"` (peso negativo).
- AC8. `bmi_category(0, 1.75) == "abaixo"` (peso zero é válido).
