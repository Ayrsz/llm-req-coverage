# req_007 — Geração de slug a partir de texto

## Descrição

O sistema deve converter um texto livre em um *slug* canônico, adequado para uso
em URLs: minúsculo, apenas com caracteres `[a-z0-9-]`, sem hifens nas pontas e
sem hifens repetidos.

## Interface

- **Módulo a importar:** `solution`
- **Assinatura:** `slugify(text: str) -> str`

```python
from solution import slugify
```

## Domínio de entrada

- `text`: string. Pode ser vazia. Pode conter qualquer caractere Unicode
  (letras acentuadas, símbolos, espaços, underscores, pontuação).

## Tratamento de acentos (decisão de projeto)

Acentos são **removidos por decomposição Unicode**: o texto é normalizado em
`NFKD` (`unicodedata.normalize`) e os caracteres combinantes (marcas de acento,
categoria `Mn`) são descartados. Assim `"á"` vira `"a"`, `"ç"` vira `"c"`,
`"ñ"` vira `"n"`. Caracteres que **não** se decompõem em uma base ASCII
alfanumérica (ex.: `"ß"`, emojis, ideogramas) são tratados como não-alfanuméricos
e portanto **removidos**. Esta escolha é simples e determinística (sem tabelas de
transliteração customizadas).

## Regras

A transformação é aplicada **nesta ordem**:

1. **normalizar acentos:** decompor em `NFKD` e descartar marcas combinantes;
2. **minúsculas:** converter todo o texto para minúsculas;
3. **separadores:** substituir cada espaço e cada underscore (`_`) por um hífen `-`;
4. **filtrar:** remover todo caractere que não esteja em `[a-z0-9-]`;
5. **colapsar:** reduzir sequências de hifens consecutivos a um único hífen;
6. **aparar:** remover hifens das duas pontas.

## Classes de equivalência

- texto alfanumérico simples → minúsculas sem alteração estrutural;
- texto com espaços/underscores → viram hífen único entre tokens;
- texto com acentos → acentos removidos, base ASCII preservada;
- texto com pontuação/símbolos → símbolos removidos;
- texto só com separadores/símbolos → `""`;
- string vazia → `""`.

## Valores-limite

- `""` → `""` (vazia);
- `"a"` → `"a"` (um caractere alfanumérico);
- `"-"` → `""` (só separador, aparado);
- `"a_b"` → `"a-b"` (underscore vira hífen);
- `"a  b"` → `"a-b"` (espaços múltiplos colapsam em um hífen).

## Entradas inválidas

Não há entrada "inválida" que lance erro: qualquer string produz um slug (no
limite, `""`). Casos degenerados:

- `"!!!"` → `""` (só símbolos);
- `"   "` → `""` (só espaços);
- `"___"` → `""` (só underscores).

## Invariantes

- O resultado contém **apenas** caracteres em `[a-z0-9-]`.
- O resultado **não** começa nem termina com `-`.
- O resultado **não** contém dois `-` consecutivos.
- A função é determinística e não lança exceção para nenhuma string de entrada.

## Critérios de aceitação

- AC1. `slugify("  Olá, Mundo! ") == "ola-mundo"` (acentos removidos, pontuação removida, pontas aparadas).
- AC2. `slugify("Hello World") == "hello-world"` (minúsculas, espaço vira hífen).
- AC3. `slugify("foo_bar") == "foo-bar"` (underscore vira hífen).
- AC4. `slugify("a---b") == "a-b"` (hifens repetidos colapsam).
- AC5. `slugify("-already-slug-") == "already-slug"` (pontas aparadas).
- AC6. `slugify("Café com Leite") == "cafe-com-leite"` (acento decomposto).
- AC7. `slugify("") == ""` (vazia).
- AC8. `slugify("!!!") == ""` (só símbolos).
- AC9. `slugify("Python 3.10") == "python-310"` (ponto removido, dígitos preservados).
