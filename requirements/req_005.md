# req_005 — Classificação de triângulo por lados

## Descrição

O sistema deve receber os comprimentos dos três lados de um triângulo (inteiros)
e classificá-lo em `"equilatero"`, `"isosceles"` ou `"escaleno"`. Se os lados
não formam um triângulo válido, retorna `"invalido"`.

## Interface

- **Módulo a importar:** `solution`
- **Assinatura:** `triangle_type(a: int, b: int, c: int) -> str`

```python
from solution import triangle_type
```

## Domínio de entrada

- `a`, `b`, `c`: inteiros, em qualquer unidade de comprimento. Devem ser
  estritamente positivos (`> 0`); qualquer lado `<= 0` torna a entrada inválida.

## Regras

1. **Validação (guarda, avaliada primeiro):** a entrada é inválida (`"invalido"`)
   se algum lado for `<= 0`, **ou** se a desigualdade triangular **estrita**
   for violada — isto é, se **não** valer simultaneamente
   `a + b > c`, `a + c > b` e `b + c > a`.
2. Um triângulo **degenerado** (igualdade, ex.: `a + b == c`) é tratado como
   **inválido** (a desigualdade é estrita, com `>`, não `>=`).
3. Se válido:
   - os três lados iguais (`a == b == c`) → `"equilatero"`;
   - exatamente dois lados iguais → `"isosceles"`;
   - os três lados distintos → `"escaleno"`.

## Classes de equivalência

- algum lado `<= 0` → `"invalido"`;
- viola desigualdade triangular (estrita) → `"invalido"`;
- três lados iguais → `"equilatero"`;
- dois lados iguais → `"isosceles"`;
- todos distintos → `"escaleno"`.

## Valores-limite

- `(1, 1, 2)` → `"invalido"` (degenerado: `1 + 1 == 2`, não `>`);
- `(2, 3, 4)` → `"escaleno"` (válido por pouco; `2 + 3 > 4`);
- `(1, 1, 1)` → `"equilatero"` (menor equilátero);
- `(0, 1, 1)` → `"invalido"` (lado zero);
- `(-1, 2, 2)` → `"invalido"` (lado negativo).

## Entradas inválidas

- Qualquer lado `<= 0`.
- Triângulo degenerado, em que a soma de dois lados é **igual** ao terceiro.
- Soma de dois lados **menor** que o terceiro.

## Invariantes

- O retorno é sempre uma das quatro strings:
  `{"invalido", "equilatero", "isosceles", "escaleno"}`.
- A classificação independe da ordem dos argumentos (simétrica em `a`, `b`, `c`).

## Critérios de aceitação

- AC1. `triangle_type(3, 3, 3) == "equilatero"`.
- AC2. `triangle_type(5, 5, 8) == "isosceles"`.
- AC3. `triangle_type(2, 3, 4) == "escaleno"`.
- AC4. `triangle_type(1, 1, 2) == "invalido"` (degenerado, desigualdade não estrita).
- AC5. `triangle_type(1, 2, 5) == "invalido"` (viola desigualdade: `1 + 2 < 5`).
- AC6. `triangle_type(0, 1, 1) == "invalido"` (lado zero).
- AC7. `triangle_type(-3, 3, 3) == "invalido"` (lado negativo).
- AC8. `triangle_type(8, 5, 5) == "isosceles"` (simetria de ordem).
