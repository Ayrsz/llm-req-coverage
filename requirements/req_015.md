# req_015 — Contador com sequência de comandos

## Descrição

O sistema deve aplicar uma sequência de comandos a um contador inteiro que
começa em `start`. Cada comando incrementa, decrementa, dobra ou zera o contador,
aplicado na ordem em que aparece. O contador **não pode ficar abaixo de 0**: se
um decremento o levaria a um valor negativo, ele permanece em `0` (piso).

## Interface

- **Módulo a importar:** `solution`
- **Assinatura:** `apply_commands(start: int, commands: list) -> int`

```python
from solution import apply_commands
```

## Domínio de entrada

- `start`: inteiro (`int`). Valor inicial do contador (assume-se `>= 0`).
- `commands`: lista de strings, aplicadas em ordem. Comandos reconhecidos:
  - `"inc"`: incrementa em 1;
  - `"dec"`: decrementa em 1;
  - `"double"`: multiplica por 2;
  - `"reset"`: zera o contador (define como 0).
  Qualquer outra string é um comando desconhecido.
- Lista vazia: nenhum comando aplicado, retorna o próprio `start`.

## Regras

1. O contador começa em `start`.
2. Para cada comando na ordem da lista:
   - `"inc"`: contador `+= 1`;
   - `"dec"`: contador `-= 1`, **mas com piso em 0** — se o resultado seria
     negativo, o contador fica em `0`;
   - `"double"`: contador `*= 2`;
   - `"reset"`: contador `= 0`;
   - comando **desconhecido**: **ignorado** (contador inalterado).
3. Invariante: o contador nunca fica abaixo de 0.
4. O resultado é um inteiro.

## Classes de equivalência

- lista vazia → retorna `start`;
- só `inc` → `start + n`;
- só `dec` com folga → `start - n`;
- `dec` levaria a negativo → fica em `0` (piso);
- `double` → dobra o valor corrente;
- `reset` → zera;
- comando desconhecido → ignorado;
- mistura de comandos → aplicados em ordem.

## Valores-limite

- `dec` quando contador `== 0` → mantém `0` (fronteira: piso);
- `dec` quando contador `== 1` → vai a `0`;
- `double` quando contador `== 0` → mantém `0`;
- lista vazia → `start`.

## Entradas inválidas

- Comando fora de `{"inc", "dec", "double", "reset"}`: ignorado (não levanta erro).
- `dec` que tornaria o contador negativo: aplicado o piso em 0 (não levanta erro).

## Invariantes

- O contador nunca fica abaixo de 0.
- Comandos desconhecidos não alteram o contador.
- A ordem dos comandos é respeitada.

## Critérios de aceitação

- AC1. `apply_commands(5, []) == 5` (lista vazia retorna start).
- AC2. `apply_commands(0, ["inc", "inc", "inc"]) == 3` (incrementos).
- AC3. `apply_commands(5, ["dec", "dec"]) == 3` (decrementos com folga).
- AC4. `apply_commands(0, ["dec"]) == 0` (fronteira: dec em 0 mantém 0).
- AC5. `apply_commands(1, ["dec", "dec"]) == 0` (piso: segundo dec não passa de 0).
- AC6. `apply_commands(3, ["double"]) == 6` (dobra).
- AC7. `apply_commands(7, ["reset"]) == 0` (zera).
- AC8. `apply_commands(4, ["spin", "inc"]) == 5` (comando desconhecido ignorado).
