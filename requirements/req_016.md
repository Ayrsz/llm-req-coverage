# req_016 — URLs mais visitadas do histórico

## Origem (documento real)

Derivado da documentação do Firefox `dataset/req_history.txt` (Places history),
que descreve, textualmente:

> *"a query folder can be created for 'My favorite pages' which contains the **20
> most frequently visited URLs** in history."*

Generalizamos o "20" para um parâmetro `n` e isolamos a parte determinística
(contagem e ranqueamento de visitas), mantendo a interface por assinatura. É um
requisito de **validade externa** (Fase 6): origem em documento real, não em
função-brinquedo sintética.

## Descrição

Dado o registro de visitas de navegação (uma entrada por visita, cada uma sendo a
URL visitada), retornar as `n` URLs mais frequentemente visitadas, em ordem
decrescente de frequência.

## Interface

- **Módulo a importar:** `solution`
- **Assinatura:** `most_visited_urls(visits: list, n: int) -> list`

```python
from solution import most_visited_urls
```

## Domínio de entrada

- `visits`: lista de strings (URLs). Cada ocorrência conta como **uma visita**.
  Pode estar vazia.
- `n`: inteiro. Quantidade de URLs a retornar. Pode ser `0` ou negativo (inválido).

## Regras

1. Conta a frequência de cada URL distinta em `visits`.
2. Retorna as URLs ordenadas por **frequência decrescente**.
3. **Desempate:** em caso de mesma frequência, vem antes a URL cuja **primeira
   visita** ocorreu antes em `visits` (ordem de primeira aparição).
4. Retorna no máximo `n` URLs (as `n` primeiras do ranqueamento).
5. Se `n <= 0`, retorna `[]`.
6. Se `n` for maior ou igual ao número de URLs distintas, retorna **todas** as
   distintas (ordenadas pelas regras 2–3).
7. O resultado **não contém repetição** (uma entrada por URL distinta).

## Classes de equivalência

- `visits` vazio → `[]` (para qualquer `n`);
- `n <= 0` → `[]`;
- `n <` nº de distintas → ranking truncado em `n`;
- `n >=` nº de distintas → todas as distintas ranqueadas;
- empate de frequência → ordem de primeira aparição.

## Valores-limite

- `n = 0` → `[]`;
- `n = 1` → só a URL mais visitada;
- `n =` nº de distintas → todas;
- `visits = []` → `[]`.

## Entradas inválidas

- `n` negativo → `[]` (invariante: nunca retorna lista de tamanho negativo).

## Invariantes

- `len(resultado) == min(max(n, 0), nº de URLs distintas)`.
- Todas as URLs do resultado pertencem a `visits`.
- Sem URLs repetidas no resultado.

## Critérios de aceitação

- AC1. `most_visited_urls(["a","b","a","c","a","b"], 2) == ["a","b"]` (a=3, b=2, c=1).
- AC2. `most_visited_urls(["a","b","a","c","a","b"], 1) == ["a"]`.
- AC3. `most_visited_urls([], 5) == []`.
- AC4. `most_visited_urls(["x","y"], 0) == []` e `most_visited_urls(["x","y"], -1) == []`.
- AC5. Desempate por primeira aparição: `most_visited_urls(["b","a","a","b"], 2) == ["b","a"]` (b e a têm freq 2; b apareceu primeiro).
- AC6. `n` maior que distintas: `most_visited_urls(["a","b","a"], 9) == ["a","b"]`.
