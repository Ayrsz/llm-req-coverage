# req_017 — Filtrar bookmarks por tag (Smart Folder)

## Origem (documento real)

Derivado da documentação do Firefox `dataset/req_bookmark.txt`, seção
*"Smart Folders and Tagging"*, que descreve textualmente:

> *"I would like to be able to sort my bookmarks into 'Smart Folders,' and have
> those folders contain bookmarks based on various criteria, including by way of
> a user-definable 'tagging' system."*

Isolamos o núcleo determinístico de uma "Smart Folder" baseada em tag: dado um
conjunto de bookmarks com tags, retornar os que possuem uma tag específica.
Requisito de **validade externa** (Fase 6): origem em documento real.

## Descrição

Filtrar uma lista de bookmarks, retornando os títulos daqueles que possuem uma
tag dada (correspondência **exata**, **sem distinção de maiúsculas/minúsculas**),
preservando a ordem original.

## Interface

- **Módulo a importar:** `solution`
- **Assinatura:** `filter_by_tag(bookmarks: list, tag: str) -> list`

```python
from solution import filter_by_tag
```

## Domínio de entrada

- `bookmarks`: lista de dicionários, cada um com as chaves `"title"` (str) e
  `"tags"` (lista de strings). Pode estar vazia.
- `tag`: string com a tag procurada. Pode ser vazia (inválida).

## Regras

1. Retorna os `"title"` dos bookmarks cuja lista `"tags"` contém `tag`.
2. A correspondência é **exata por tag** (não é substring): a tag `"py"` não
   casa com `"python"`.
3. A correspondência é **case-insensitive**: `"Trabalho"` casa com `"trabalho"`.
4. A ordem do resultado preserva a ordem de `bookmarks`.
5. Se `tag` for vazia (`""`), retorna `[]`.
6. Nenhum bookmark correspondente → `[]`.

## Classes de equivalência

- `tag` vazia → `[]`;
- `bookmarks` vazio → `[]`;
- tag presente em alguns → títulos correspondentes na ordem original;
- tag com caixa diferente → casa mesmo assim (regra 3);
- tag que é substring de outra tag → **não** casa (regra 2).

## Valores-limite

- `tag = ""` → `[]`;
- bookmark com `"tags": []` → nunca casa;
- tag igual a uma tag existente exceto pela caixa → casa.

## Entradas inválidas

- `tag` vazia → `[]` (invariante: sem tag não há filtro).

## Invariantes

- O resultado é uma sublista (em ordem) dos títulos de `bookmarks`.
- `len(resultado) <= len(bookmarks)`.

## Critérios de aceitação

Considere:
```python
bms = [
    {"title": "Python docs", "tags": ["dev", "python"]},
    {"title": "Receitas",    "tags": ["casa", "comida"]},
    {"title": "Flask",       "tags": ["dev", "Python"]},
]
```

- AC1. `filter_by_tag(bms, "dev") == ["Python docs", "Flask"]` (ordem preservada).
- AC2. `filter_by_tag(bms, "python") == ["Python docs", "Flask"]` (case-insensitive: casa "python" e "Python").
- AC3. `filter_by_tag(bms, "py") == []` (correspondência exata, não substring).
- AC4. `filter_by_tag(bms, "") == []` (tag vazia).
- AC5. `filter_by_tag([], "dev") == []` (sem bookmarks).
- AC6. `filter_by_tag(bms, "comida") == ["Receitas"]`.
